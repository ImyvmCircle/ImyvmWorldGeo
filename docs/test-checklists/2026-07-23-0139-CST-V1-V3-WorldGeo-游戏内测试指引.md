# 2026-07-23 01:39 CST V1-V3 WorldGeo 游戏内测试指引

## 适用范围

本指引用于验证 IMYVMWorldGeo 26.2-1.5.5 对 Imyvm-Design 正文 V1-V3 意图的游戏内落地状态。范围包括 Region、Scope、SubSpace 三层空间、标签、设置解析、自然时间、测试周期、行为事件、周期统计、空间快照、设置摘要、空间消息、地理画像和 OP 调试入口。Community 与 Adventure 的城邦、荒野、奖励、战争、财政业务不属于本次 WorldGeo 验收范围。

## 测试前准备

- 使用 Minecraft 26.2、Fabric Loader 0.19.3、Java 25、Hoki 1.1.6、IMYVMWorldGeo 26.2-1.5.5。
- 启动服务端并确认日志出现 `imyvmworldgeo initialized successfully` 与 `Done`。
- 准备至少两个在线账号：一个 OP，一个普通玩家。普通玩家用于验证可见性、权限、消息触达和玩家枚举。
- 准备一块平坦区域，建议坐标跨度不小于 80x80，避免与旧 Region 重叠。
- 测试前备份 `world/imyvm_world_geo` 或对应存档目录下的 WorldGeo 数据文件。

## V1 空间与设置验收

1. 创建 Region：OP 执行 `/imyvmWorldGeo select start rectangle`，选择两个角点后执行 `/imyvmWorldGeo create rectangle v155_region`。预期 Region 创建成功，`/imyvmWorldGeo query v155_region` 能显示基础信息。
2. 创建 Scope：在 Region 内选择较小矩形，执行 `/imyvmWorldGeo addScope rectangle v155_region v155_scope`。预期 Scope 成功归属到 Region，查询 Region 能看到 Scope。
3. 创建 SubSpace：使用当前公开命令或调试入口创建 `v155_subspace`，其形状必须完全落在 `v155_scope` 内。预期保存成功，重启后仍存在。
4. 负向验证 SubSpace 归属：尝试创建超出父 Scope 或与兄弟 SubSpace 冲突的 SubSpace。预期操作失败，已有 Region、Scope、SubSpace 不被部分写入。
5. 验证三层解析：分别站在 Region-only、Scope 内但 SubSpace 外、SubSpace 内三个位置，观察 actionbar、查询输出或调试输出。预期解析优先级为 SubSpace > Scope > Region。
6. 验证标签：为 SubSpace 添加 string tag 与 keyed tag，再删除其中一个。预期查询或快照能反映新增、覆盖和删除，重启后结果一致。
7. 验证设置优先级：在 Region、Scope、SubSpace 设置同一 permission/rule/effect 的不同值。玩家站在三类位置时，预期读取结果按 SubSpace、Scope、Region、默认值顺序生效。
8. 验证持久化回读：重启服务端后重复查询 Region、Scope、SubSpace、标签、设置。预期 ID、名称、父子关系、形状、标签、设置均保持一致。

## V2 时间、事件与统计验收

1. 自然时间快照：在服务端或 addon 测试入口调用 `RegionDataApi.getTimeSnapshot(level)` 与 `getRealTimeSnapshot("Asia/Taipei")`。预期返回游戏时间、日期、月相、季节提示和指定时区实时字段。
2. 周期 ID：调用 `getCurrentNaturalPeriodIds()`。预期包含 `HOUR`、`DAY`、`WEEK`、`MONTH`、`YEAR`，且 periodId 格式稳定。
3. 测试周期：通过 OP 调试命令进入短周期测试模式，等待至少两次周期切换。预期周期回调只对完成的周期触发，不重复触发当前周期。
4. 重启补偿：在测试周期跨越后重启服务端。预期漏掉的已完成周期被枚举补偿，当前未完成周期不被误结算。
5. 行为事件字段：在 Region、Scope、SubSpace 内分别放置方块、破坏方块、移动停留、产生击杀或伤害事件。预期 recent behavior events 包含 eventId、type、playerUuid、playerName、dimensionId、坐标、unixMillis、region/scope/subspace id 与 name、spaceLevel、objectId、targetId、quantity、source。
6. 周期统计查询：对相同 periodKind 和 periodId 调用 `queryBehaviorStats`。预期能按 behaviorType、Region、Scope、SubSpace、playerUuid、objectId、targetId 过滤。
7. 类型化统计：分别调用 `queryBlockDelta`、`queryResidence`、`queryEntityCombat`、`queryOnlineTime`。预期方块净变化、驻留毫秒、战斗计数、在线或 AFK 毫秒与刚才操作方向一致。
8. 容量与稳定性：连续触发多次行为事件后查看日志。预期无主线程异常、无无界日志刷屏，统计文件可继续读写。

## V3 快照、消息与调试验收

1. 空间快照：调用 `getRegionSpaceSnapshot`、`getScopeSpaceSnapshot`、`getSubSpaceSnapshot`。预期返回 type、id、name、dimensionId、area、父级 ID 与名称、子数量、标签、statsVersion、dominantBiomeId、entryMessageEnabled、entryMessageConfigured、mapColorSuggestion、settingSummary、displayName、shapeType、shapeParameters。
2. 列表快照：调用 `listScopeSnapshots(regionId)` 和 `listSubSpaceSnapshots(scopeId)`。预期只返回目标父空间下的子空间快照。
3. 设置摘要：调用 `listRegionSettingSummaries`、`listScopeSettingSummaries`、`listSubSpaceSettingSummaries`，分别传入 `PUBLIC` 和 `OP_DEBUG`。预期 PUBLIC 不泄漏 OP-only 信息，OP_DEBUG 包含调试可见字段。
4. 地理画像：调用 Region、Scope、SubSpace 的 geographic profile 与 snapshot API。预期返回 biome/地表/结构/难度相关结果；当候选 chunk 超限或维度不可用时返回明确状态而非崩溃。
5. 玩家枚举：让两个玩家分别站在 Region、Scope、SubSpace 内外，调用 `listPlayersInRegion`、`listPlayersInScope`、`listPlayersInSubSpace`。预期只返回在线且位于目标空间内的 UUID，缓存未命中时仍能即时解析。
6. 空间消息：分别调用 Component overload 与 `messageKey, vararg args` overload 的 `sendRegionSpaceMessage`、`sendScopeSpaceMessage`、`sendSubSpaceMessage`。预期只有目标空间内玩家收到消息；缺失翻译 key 时返回 0 并记录错误。
7. 位置 actionbar：调用 `toggleLocationActionBar`、`setLocationActionBarVisible`、`isLocationActionBarVisible`。预期显示状态可切换、可查询，并与旧 `toggleActionBar` 行为一致。
8. OP 调试视图：OP 调用 `openSpaceDebugView` 的 Region、Scope、SubSpace overload。预期返回 1 或发送调试信息；普通玩家调用预期返回 0 且不显示 OP 调试内容。
